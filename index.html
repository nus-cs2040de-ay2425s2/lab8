SSSP: Dijkstra's algorithm

Without Negative Edge Weights

1. Dense Graphs: Naive Dijkstra Algorithm

For dense graphs, where the number of edges is close to the square of the number of vertices, the Naive Dijkstra Algorithm can be used. It employs an array to find the shortest paths from a source node.

  • Time Complexity: O(V2), where V is the number of vertices.

import java.util.Arrays;


public class NaiveDijkstra {
    // you can also define public static final int INF = 1000000000;
    static final int N = 10010;
    static int[][] g = new int[N][N];  // Stores each edge
    static int[] dist = new int[N];    // Stores the shortest distance from source (1) to each node
    static boolean[] st = new boolean[N];  // Stores whether the shortest path to a node is determined
    static int n;  // Number of nodes


    public static int dijkstra() {
        Arrays.fill(dist, Integer.MAX_VALUE / 2);
        dist[1] = 0;


        for (int i = 0; i < n - 1; i++) {
            int t = -1;
            for (int j = 1; j <= n; j++) {
                if (!st[j] && (t == -1 || dist[t] > dist[j])) {
                    t = j;
                }
            }


            for (int j = 1; j <= n; j++) {
                // Only process if there is an actual edge (not infinity/no connection)
                if (g[t][j] < Integer.MAX_VALUE / 2 && g[t][j] > 0) {
                    dist[j] = Math.min(dist[j], dist[t] + g[t][j]);
                }
            }


            st[t] = true;
        }


        return dist[n] == Integer.MAX_VALUE / 2 ? -1 : dist[n];
    }
}

 

2. Sparse Graphs: Dijkstra Algorithm with Heap Optimization

For sparse graphs, where the number of edges is much less than the square of the number of vertices, Dijkstra's Algorithm with a min-heap or priority queue is used to optimize the selection of the next node to process.

  • Time Complexity: O((V+E)∗logV), where E is the number of edges.

import java.util.*;


public class DijkstraHeapOptimized {
    static final int N = 10010;
    static int[] h = new int[N], w = new int[N], e = new int[N], ne = new int[N], dist = new int[N];
    static boolean[] st = new boolean[N];
    static int idx, n;


    public static void initialize() {
        Arrays.fill(h, -1);
        Arrays.fill(dist, Integer.MAX_VALUE / 2);
        idx = 0;
    }


    public static void addEdge(int a, int b, int weight) {
        e[idx] = b;
        w[idx] = weight;
        ne[idx] = h[a];
        h[a] = idx++;
    }


    public static int dijkstra() {
        PriorityQueue<int[]> heap = new PriorityQueue<>(Comparator.comparingInt(a -> a[0]));
        dist[1] = 0;
        heap.offer(new int[]{0, 1});


        while (!heap.isEmpty()) {
            int[] t = heap.poll();
            int ver = t[1], distance = t[0];


            if (st[ver]) continue;
            st[ver] = true;


            for (int i = h[ver]; i != -1; i = ne[i]) {
                int j = e[i];
                if (dist[j] > distance + w[i]) {
                    dist[j] = distance + w[i];
                    heap.offer(new int[]{dist[j], j});
                }
            }
        }


        return dist[n] == Integer.MAX_VALUE / 2 ? -1 : dist[n];
    }
}

SSSP: Bellman-Ford's algorithm

With Negative Edge weights

Bellman-Ford Algorithm

The Bellman-Ford Algorithm can handle graphs with negative edge weights (but not negative cycles). It relaxes all edges n - 1 times, where n is the number of vertices.

  • Time Complexity: O(V∗E), where V is the number of vertices and E is the number of edges.

import java.util.Arrays;


public class BellmanFord {
    static final int N = 10010, M = 100010;
    static int n, m;
    static int[] dist = new int[N];
    static Edge[] edges = new Edge[M];


    static class Edge {
        int a, b, w;


        public Edge(int a, int b, int w) {
            this.a = a;
            this.b = b;
            this.w = w;
        }
    }


    public static int bellmanFord() {
        Arrays.fill(dist, Integer.MAX_VALUE / 2);
        dist[1] = 0;


        for (int i = 0; i < n - 1; i++) {
            for (int j = 0; j < m; j++) {
                int a = edges[j].a, b = edges[j].b, w = edges[j].w;
                if (dist[a] != Integer.MAX_VALUE / 2 && dist[b] > dist[a] + w) {
                    dist[b] = dist[a] + w;
                }
            }
        }


        return dist[n] > Integer.MAX_VALUE / 2 / 2 ? -1 : dist[n];
        // Previously, we use Integer.MAX_VALUE / 2 as a safe bound, but why we use Integer.MAX_VALUE / 2 / 2 in this case?
    }
}

APSP: Floyd–Warshall algorithm

All-pairs Shortest Path

Floyd-Warshall Algorithm - Dynamic Programming based

The Floyd-Warshall Algorithm is used to find the shortest paths between all pairs of vertices in a graph. It works for graphs with both positive and negative edge weights, but no negative cycles.

  • Time Complexity: O(V3), where V is the number of vertices.

import java.util.Arrays;


public class FloydWarshall {
    static final int N = 110;
    static int[][] d = new int[N][N];
    static int n;


    public static void floyd() {
        for (int k = 1; k <= n; k++) {
            for (int i = 1; i <= n; i++) {
                for (int j = 1; j <= n; j++) {
                    d[i][j] = Math.min(d[i][j], d[i][k] + d[k][j]);
                }
            }
        }
    }


    public static void initialize() {
        for (int i = 1; i <= n; i++) {
            for (int j = 1; j <= n; j++) {
                if (i == j) {
                    d[i][j] = 0;
                } else {
                    d[i][j] = Integer.MAX_VALUE / 2;
                }
            }
        }
    }
}

Following is an application of Bellman-Ford algorithm:

Problem Description

Given a directed graph with n nodes and m edges, where edges may have negative weights, find the shortest path from node 1 to node n that uses at most k edges. If there is no such path, output "impossible".

Note: The graph may contain negative weight cycles.

Input

  • The first line contains three integers n, m, k.

  • The following m lines contain three integers x, y, z, representing a directed edge from node x to node y with weight z.

Output

  • Output the shortest distance from node 1 to node n using at most k edges. If no such path exists, output "impossible".

Example

Input:

3 3 1
1 2 1
2 3 1
1 3 3

Output:

3

 

import java.util.Arrays;
import java.util.Scanner;


/**
 * Problem: Shortest Path with Limited Edges
 */


class Edge {
    int a, b, c;


    public Edge(int a, int b, int c) {
        this.a = a;
        this.b = b;
        this.c = c;
    }
}


public class LimitedEdgesBellmanFord {
    static final int N = 510;
    static final int M = 10010;
    static final int INF = Integer.MAX_VALUE / 2;


    static Edge[] edges = new Edge[M];
    static int[] dist = new int[N];
    static int[] last = new int[N];
    static int n, m, k;


    /**
     * Bellman-Ford algorithm with limited edges
     * 
     * Given a directed graph with n nodes and m edges, where edges may have negative weights,
     * find the shortest path from node 1 to node n that uses at most k edges. If there is no such path,
     * output "impossible".
     * 
     * Note: The graph may contain negative weight cycles.
     */
    public static void bellmanFord() {
        Arrays.fill(dist, INF);
        dist[1] = 0;


        for (int i = 0; i < k; i++) {
            System.arraycopy(dist, 0, last, 0, dist.length);
            for (int j = 0; j < m; j++) {
                Edge e = edges[j];
                if (last[e.a] < INF) {
                    dist[e.b] = Math.min(dist[e.b], last[e.a] + e.c);
                }
            }
        }
    }


    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);
        n = scanner.nextInt();
        m = scanner.nextInt();
        k = scanner.nextInt();


        for (int i = 0; i < m; i++) {
            int a = scanner.nextInt();
            int b = scanner.nextInt();
            int c = scanner.nextInt();
            edges[i] = new Edge(a, b, c);
        }


        bellmanFord();


        if (dist[n] > INF / 2) {
            System.out.println("impossible");
        } else {
            System.out.println(dist[n]);
        }
    }
}
Analysis

This problem is a variant of the Bellman-Ford algorithm where we limit the number of edges that can be used.

Chaining Issue: Due to the nature of the algorithm, each update in an iteration must be based on the shortest path considering exactly one additional edge compared to the previous iteration. If we directly use dist[e.b] = min(dist[e.b], dist[e.a] + e.c);, we cannot guarantee that dist[e.a] has not been updated within the current iteration. If it has been updated, and dist[e.b] > dist[e.a] + e.c, it would mean that the current update for node b effectively considers more than one edge, leading to incorrect results. By using dist[e.b] = min(dist[e.b], last[e.a] + e.c);, we ensure that the value of a used to update b is from the previous iteration, thus maintaining the correctness of considering only one additional edge at each iteration.
